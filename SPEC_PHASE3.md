## 1. 冲动拦截器（Impulse Interceptor）

### 概述
在用户「拿起手机准备下单」的关键时刻介入，用许愿池进度拦截冲动，把购买入口变成冲动过滤入口。

### 触发入口
主页常驻一个大 CTA 按钮，与现有对话框并列，文案：**「我现在想买……」**

> 不替换现有对话框，两个入口职责分离：
> - 主对话框：录入（记账、忍住了、冲动记录等）
> - 拦截器按钮：专门拦截「准备去买」的冲动时刻

### 交互流程

```
点击「我现在想买……」
    ↓
弹出极简输入框：「你现在想买什么？」
    ↓
用户输入（如「想买那双 Nike 699 块」）
    ↓
AI 解析，不跳确认卡
    ↓
全屏显示许愿池代价画面（见下）
    ↓
用户选择出口
```

### 许愿池代价画面（关键）
全屏显示，停留约 3 秒，内容：

```
你现在的许愿池目标是：
AirPods Pro
已攒 ¥340 / ¥1299

买这双鞋 = 许愿池退回到 ¥0
```

配动画：进度条从当前值**往回倒**至 0，视觉上像打碎存钱罐。

> 若无许愿池目标，则跳过此画面，直接走冲动过滤流程。

### 两个出口

```
[还是想买，记录冲动]     [算了，我忍了]
```

- **还是想买** → 走冲动过滤流程（72h 冷静期），正常记录
- **算了，我忍了** → 弹出「忍住了多少钱？」（AI 已预填金额，用户确认）→ 自动累积进许愿池 → 进度条往上涨，触发仪式感动画

### 设计原则
把抽象的「我在乱花钱」变成具体的代价：许愿池倒退的那一刻，比任何文字提醒都有感知。

### 涉及文件
- `src/components/home/ImpulseInterceptorButton.tsx` — 新建，主页大 CTA
- `src/components/home/WishPoolConsequenceScreen.tsx` — 新建，全屏代价画面
- `src/store/wishpool.ts` — 复用 `addSavings`，无新逻辑
- `src/pages/Home.tsx` — 新增按钮入口

## 2. 预算感知系统（Budget Awareness System）

### 概述
把「已花多少」改为「还能花多少」，配合记账瞬间的数字滚动，让每一笔消费都有即时代价感。

### 2.1 主页大数字重设计

**现在：**
```
可支配消费 ¥1240 / ¥2000
```

**改为：**
```
还能花
¥760
```

一个大数字，字号尽量大，占主页上方核心位置。

#### 三种状态

| 状态 | 条件 | 视觉 |
|------|------|------|
| 宽裕期 | 剩余 > 50% | 正常黑色，无压迫感 |
| 警戒期 | 剩余 20%–50% | 数字琥珀色 + 下方小字「距月底还有 X 天」 |
| 危险期 | 剩余 < 20% | 数字红色放大 + 小字「平均每天只剩 ¥X」 |
| 超支后 | 已超支 | 显示「已超支 ¥XX」，红色加粗，背景轻微变色 |

> 危险期换算成「日均剩余」，比百分比更有损失感。
> 距月底天数用 `daysUntilMonthEnd()` 计算，不说教，让用户自己算。

### 2.2 记账确认瞬间的「代价闪烁」

用户点确认记账，卡片消失的同时，主页大数字触发**滚动动画**：

```
¥761 → ¥733
```

- 动画形式：数字像老虎机倒转，耗时约 0.8 秒
- 配极轻触感震动（设备支持时）
- 安静，无弹窗，无提示文字

#### 三种金额的差异化反馈

| 金额 | 反馈 |
|------|------|
| 小额（< ¥50） | 数字安静滚动，无其他变化 |
| 中额（¥50–¥200） | 数字滚动 + 短暂琥珀色闪烁后恢复 |
| 大额（> ¥200） | 数字滚动 + 变色 + 小字淡入：「这笔占本月剩余的 X%」，2 秒后自动消失 |

### 完整感知回路

```
平时看主页 → 感知剩余额度（还能花 ¥761）
         ↓
每次消费确认 → 数字滚动减少（→ ¥733）
         ↓
进入警戒期 → 数字变琥珀 + 距月底天数
         ↓
进入危险期 → 数字变红 + 日均换算
         ↓
超支 → 已超支 ¥XX 常驻压在那里
```

每一步都有感知，没有任何一步在说教。

### 涉及文件
- `src/components/home/BudgetCard.tsx` — 改显示逻辑（剩余额度大数字 + 三种状态）
- `src/components/home/BudgetCard.tsx` — 新增数字滚动动画（`useCountDown` / CSS transition）
- `src/lib/budget.ts` — 新增 `daysUntilMonthEnd()` / `dailyRemaining()` 工具函数
- 数据层零改动

## 3. 月度复盘故事（Monthly Story）

### 概述
每月生成一段 AI 叙事文字，把用户当月的消费数据变成一个有温度的故事，配内嵌对话支持追问。

### 3.1 故事生成

#### 触发时机
每月 1 号自动生成上月故事，主页推送查看卡片。用户也可主动进入复盘页触发。

#### 格式示例

```
3月，你花了 ¥3,240。

大额消费 3 笔，2 笔觉得值，1 笔后悔——
那件毛衣你说「穿了一次」。

忍住了 6 次，攒进许愿池 ¥340，
AirPods Pro 进度 26%。

冲动过滤拦住了 4 件，你最终只买了 1 件。

本月人格：❄️ 季节冲动型
换季买了 3 件，2 件后悔了。
```

#### 段落生成逻辑

**必写段落（每月都有）**
- 总支出 + 和上月对比（多/少了多少）
- 可支配消费是否超支

**条件段落（有数据才写，无则跳过）**
- 有后悔复盘 → 点名那笔，引用用户填的「用了多久」
- 有许愿池记录 → 忍住次数 + 进度变化
- 有冲动过滤记录 → 拦住几件 / 最终买了几件
- 有 streak → 连续自律天数

**结尾永远一句话，不说教，略带自嘲：**
- 「总体来说，这个月你对自己还算诚实。」
- 「那件毛衣的钱，下次留给许愿池吧。」
- 「你忍住的次数比花钱的次数多，这不常见。」

> AI 只说数据里有的事，不编造，不推断。

#### 与 SPEC2 消费人格报告的关系
合并展示：故事是过程，人格标签是结论，同一张卡片里呈现。

### 3.2 内嵌对话

故事卡片底部不是「关闭」按钮，而是一个输入框：

```
___________________________
你可以问我任何关于这个月的事
```

#### 对话能回答的问题类型

**追问具体消费**
```
用户：那件毛衣为什么后悔？
AI：你买的时候说「冬季必备」，
    复盘时填的是「穿了一次，一般」。
    购买到复盘间隔 28 天。
```

**追问消费规律**
```
用户：我经常后悔买衣服吗？
AI：近 3 个月，服饰类后悔率 67%——
    3 笔里有 2 笔你觉得不值。
    冲动过滤拦住的那几件你都没后悔。
```

**追问许愿池**
```
用户：我什么时候能买到 AirPods？
AI：按过去 2 个月平均忍住速度，
    大概还需要 6 周。
    如果每周多忍一顿外卖，可以提前 2 周。
```

**追问预算建议**
```
用户：我下个月预算应该怎么设？
AI：你 3 月可支配超支 ¥240，主要在外卖（¥680）。
    建议下月外卖单独设上限，
    整体可支配额度维持 ¥2000 不变。
```

#### 对话设计原则
- 每次月度故事对话是**独立 session**，只能问「这个月」的事
- 不是全局 AI 助手，是**这份报告的注脚**
- 入口在故事卡片内，不在主对话框
- 两个对话框职责分离：主对话框 = 录入，故事对话框 = 复盘

### 涉及文件
- `src/lib/generateMonthlyStory.ts` — 新建，聚合各 store 数据 + 调用 AI 生成故事
- `src/components/review/MonthlyStoryCard.tsx` — 新建，故事展示 + 内嵌对话组件
- `src/store/review.ts` — 新增月度故事存储（存入 Supabase，不重复生成）

## 4. 执行层重设计（Execution Layer Redesign）

### 概述
把执行层从「防拖延倒计时器」升级为「临阵决策指挥室」，AI 全程参与但用户主导。

### 4.1 两种决策模式

#### ⚡ 快速决策模式（现有逻辑，保留）
- 适合：衣服、日用品、外卖、小额消费（< ¥500）
- 倒计时 15min（可调），时间到强制决策
- 逻辑：你已经想好买什么了，只差一个截止点

#### 🔍 研究模式（新增）
- 适合：电子产品、大额耐用品（> ¥500）
- 无倒计时，改为 **AI 生成任务清单**

```
买之前你需要：
☐ 看一篇评测
☐ 比较 3 个选项
☐ 确认退换货政策
☐ 查闲鱼二手价

全部勾完才能「确认购买」
```

> 清单内容 AI 根据品类预填，用户可删减。
> 点「开始研究」记录开始时间，全部勾完后显示「你研究了 X 分钟，现在可以下单了」。

#### 模式自动判断
进入执行层时 AI 自动判断，用户可手动切换：
- 估价 > ¥500 **或** 品类 = 电子产品 → 研究模式
- 其他 → 快速决策模式

### 4.2 开场决策简报

进入执行层后，AI 自动生成简报，用户 10 秒读完，0 输入成本：

```
你要买：冬季外套
预算上限：¥800（本月还能花 ¥1,240）

你的历史：
上次买外套是 8 个月前，复盘说「值」
服饰类后悔率：33%（3笔里1笔后悔）

信任品牌：优衣库(9) · 无印(7) · COS(6)

你的 SOP 提醒你：
裤子线下试，上衣优先品牌背书
贵的好牌子去闲鱼找二手

本次建议：快速决策模式 · 15min
```

#### 简报数据来源
| 数据项 | 来源 |
|--------|------|
| 预算上限 | 本月剩余可支配额度 |
| 历史购买 | `transactions` 表同品类记录 |
| 后悔率 | `review_results` 同品类统计 |
| 信任品牌 | `brand_library` 按 weight 排序 |
| SOP 提醒 | `sop_rules` 相关条目 |

### 4.3 决策对话（AI 决策助手）

研究模式 / 快速模式均可用，任务清单下方常驻输入框：

```
___________________________
把你的纠结丢给我
```

#### 三种典型场景

**两个选项比较**
```
用户：索尼WH-1000XM5 和 AirPods Max 我选哪个
AI：你的许愿池目标是 AirPods Pro，预算 ¥1299。
    XM5 现在 ¥1999，Max ¥2999，都超了。
    
    如果非要选：
    降噪和续航 → XM5
    和苹果设备无缝切换 → Max
    
    但你已经在攒 AirPods Pro 了，
    建议先把许愿池跑完再看这个决定。
```

**粘贴链接或上传截图**
```
用户：[上传两件衬衫截图]
AI：左边聚酯纤维 68%，右边纯棉。
    价格差 ¥80，材质差距明显。
    按你的 SOP「上衣优先有品牌背书的」，
    推荐右边。
```

**直接要结论**
```
用户：帮我做决定，买左边还是右边
AI：买右边。
    理由：品牌在你的品牌库里，权重 8/10。
    左边没有记录。
```

#### AI 可用上下文
执行层 AI 拥有完整上下文，给出的是针对「这个用户此刻处境」的建议，不是通用建议：
- 当前品类和估价
- 用户的 SOP 规则
- 品牌库权重
- 许愿池目标和当前进度
- 本月剩余预算
- 同品类历史购买和复盘结果

### 4.4 收尾总结

购买确认后，不只是跳记账，显示一张收尾卡片：

```
你用了 23 分钟选定了优衣库的羊毛外套 ¥699。
比上次买外套快了 12 分钟。
已加入 30 天复盘队列。
```

让用户感觉「这次决策是认真的」，不是冲动。

### 完整节奏

```
进入执行层
    ↓
AI 生成决策简报（自动，5秒内）
    ↓
用户开始找货（快速模式 / 研究模式）
    ↓
纠结时 → AI 决策对话
    ↓
决定了 → 确认购买
    ↓
AI 收尾总结 → 触发记账 + 复盘任务
```

**第二批（中等改动）**
3. 执行层重设计（简报 + 决策对话 + 收尾总结）

**第三批（依赖数据积累）**
4. 月度复盘故事（需要有一定量的复盘数据才好看）
