## 1. 代价视角（Cost Perspective）

### 概述
在待购清单、冲动过滤卡片上显示「代价」标签，将金额换算成用户有感知的现实代价，强化决策痛感。

### 用户身份模式
设置页新增「身份」选项：

**🎓 学生模式**
- 输入项：月生活费、月伙食费预算
- 换算逻辑：
  - 伙食天数 = 金额 ÷ (月伙食费 ÷ 30)，显示「X 天伙食费」
  - 生活费占比超过 30% 时显示「本月生活费的 X%」

**💼 工作党模式**
- 输入项：月薪、日工作时长
- 换算逻辑：
  - 时薪 = 月薪 ÷ 21.75 ÷ 日工作时长
  - 显示「工作 X 小时」，超过 8 小时换算成「X 天工作白干了」

### 许愿池锚点（两种模式通用，优先显示）
- 自动读取当前许愿池目标和进度
- 显示「许愿池退回到 X%」
- 若无许愿池目标则不显示此条

### 显示规则
每个商品卡片最多显示 2 条代价，优先级：
1. 许愿池锚点（永远第一）
2. 伙食天数 / 工作时长

文案风格：克制、不带感叹号、冷静陈述。示例：
```
「代价」许愿池退回到 32%
「代价」11 天伙食费
```

### 数据存储
`user_settings` 表新增字段：
- `identity`: 'student' | 'worker'
- `monthly_income`: number（月生活费 / 月薪）
- `monthly_food_budget`: number（月伙食费，学生模式）
- `daily_work_hours`: number（日工作时长，工作党模式）

### 涉及文件
- `src/pages/Settings.tsx` — 新增身份设置区块
- `src/store/userSettings.ts` — 新增字段
- `src/components/wishlist/WishlistItem.tsx` — 显示代价标签
- `src/components/impulse/ImpulseCard.tsx` — 显示代价标签
- `src/lib/costPerspective.ts` — 新建，换算逻辑

## 2. 订阅管理（Subscription Tracker）

### 概述
订阅是刚需性固定支出，归入基础支出分类，每月自动生成记账记录，临期前提醒。

### 数据结构
新建 `subscriptions` 表：
```sql
id, user_id, name, amount, billing_day（每月几号扣）,
category（流媒体/工具/出行/其他）, is_active, created_at
```

### 交互方式
- 对话框自然语言录入：「我有百度网盘会员每月 26 号扣 26 块」→ AI 解析添加
- 设置页手动管理列表

### 自动化逻辑
- 每月 billing_day 当天自动在 `transactions` 表生成一条记录，分类归入「基础支出 → 订阅」
- 扣款日前 3 天，主页推送提醒卡片：「明天将扣除 百度网盘 ¥26」

### 预算联动
订阅总额在预算页基础支出进度条中自动计入，用户无需手动记账。

### UI 位置
记账页 / 设置页新增「订阅管理」入口，展示所有订阅列表，支持增删改。

## 3. 同类替代提醒（Duplicate Detection）

### 概述
加入待购清单时，AI 扫描历史交易记录，发现同类物品自动提醒。

### 触发时机
用户将商品加入待购清单（wishlist）时，后台异步检测。

### 检测逻辑
- AI 对比新商品与近 12 个月交易记录里同二级分类的商品
- 相似度判断：同品类 + 功能重叠（如"黑色外套"与"深色风衣"）
- 超过 2 件同类时触发提醒

### 提醒文案示例
```
你已经有 3 件黑色外套（最近一件购于 2 个月前）
确定还需要这件吗？
```

提供两个按钮：「我确实需要」/ 「算了，不买了」

### 涉及文件
- `src/store/wishlist.ts` — add 方法里触发检测
- `src/lib/duplicateDetect.ts` — 新建，检测逻辑
- `src/components/wishlist/DuplicateWarningCard.tsx` — 新建

## 4. 值不值反查（Worth It？）

### 概述
待购清单里每件商品可触发 AI 评估，根据品类差异化判断逻辑。

### 品类差异化逻辑

**🛒 标准品（生活用品 / 电子产品）**
- AI 联网搜索当前价格对比、历史低价
- 结论直接：「京东现价偏高，拼多多同款便宜 40 元」

**👗 溢价品（服饰 / 包 / 鞋）**
- 用户可上传商品图片，AI 识别面料/材质
- 评估维度：材质性价比、品牌溢价合理性、二手保值率、同价位替代建议
- 结论是建议性的：「这个价位聚酯纤维性价比偏低，同价位可考虑 XX」

**🎭 体验品（演出 / 旅行 / 餐厅）**
- 无价格参照系，改为预算影响评估
- 结论：「占本月可支配预算的 34%，建议冷静期考虑」

### 触发方式
待购清单每个商品卡片右下角「值不值？」按钮，点击后展示 AI 评估卡片。

### 涉及文件
- `src/components/wishlist/WorthItCard.tsx` — 新建
- `src/lib/worthItAnalysis.ts` — 新建，品类判断 + AI 调用逻辑

## 5. 买了后悔榜（Regret Board）

### 概述
每月自动统计复盘中标记「后悔」的购买，月底生成榜单，数字化浪费。

### 数据来源
`review_results` 表中 `worth_it = false` 的记录。

### 展示内容
- 本月后悔购买列表（商品名、金额、购买日期）
- 总计浪费金额
- 最常后悔的分类（比如「你的冲动消费重灾区是：服饰」）

### 触发时机
每月最后一天，或用户主动进入复盘页查看。

### 文案风格
自嘲、不说教，用数字说话：
```
本月后悔榜
共 3 笔 · ¥487 打了水漂

① 那件穿了一次的毛衣   ¥239
② 冲动买的键盘          ¥168
③ 根本不好喝的咖啡豆    ¥80

你的消费重灾区：服饰
```

### 涉及文件
- `src/pages/Review.tsx` — 新增后悔榜区块
- `src/store/review.ts` — 新增月度后悔统计查询

## 6. 消费人格报告（Spending Persona）

### 概述
每月分析支出模式，给用户贴一个消费人格标签，附简短改进建议。

### 人格类型

| 标签 | 判断条件 | 描述 |
|------|---------|------|
| 🐿️ 囤货松鼠型 | 同类物品反复购买，触发同类提醒 ≥3 次 | 买东西喜欢囤，家里总有同款 |
| 🎭 仪式感驱动型 | 消费集中在节日/纪念日前后 | 需要一个理由才能放心花钱 |
| ❄️ 季节冲动型 | 消费在换季月份集中爆发 | 每次换季都是大采购 |
| 💨 即时满足型 | 冷静期放弃但当月消费总额不降 | 忍住了这个，马上买了那个 |
| 🧘 自律守护型 | streak ≥ 14 天，后悔榜为空 | 本月消费克制，值得表扬 |

### 生成时机
每月 1 号自动生成上月报告，主页推送查看提醒。

### 展示形式
一张卡片：人格标签 + emoji + 一句话描述 + 一条改进建议。轻量，不说教。

### 涉及文件
- `src/lib/personaAnalysis.ts` — 新建，分析逻辑
- `src/components/review/PersonaCard.tsx` — 新建

## 7. 许愿池彩蛋动画（Wish Pool Milestone）

### 概述
许愿池进度达到 25% / 50% / 75% / 100% 时触发松鼠庆祝动画。

### 动画设计
- 25%：松鼠找到一颗栗子，开心跳跳
- 50%：松鼠抱着栗子跑
- 75%：松鼠把栗子放进树洞
- 100%：树洞满了，松鼠在上面转圈庆祝 + 彩纸飘落

### 触发逻辑
每次 `addSavings` 后检查进度是否越过里程碑，每个里程碑只触发一次。

### 涉及文件
- `src/components/wishpool/MilestoneAnimation.tsx` — 新建
- `src/store/wishpool.ts` — 新增里程碑检测逻辑

## 8. Streak + 成就系统（Achievements）

### Streak 定义
连续自然日内无新增冲动消费记录（impulse）即为「自律」。
- 破功条件：当天新增任意一条 impulse 记录
- 每日 00:00 检查，streak 天数 +1 或归零

### 成就列表

| 成就 | emoji | 触发条件 |
|------|-------|---------|
| 第一颗栗子 | 🌰 | 第一次「忍住了」 |
| 铁石心肠 | 🧘 | 累计忍住金额 ≥ 5000 元 |
| 冷静如冰 | ❄️ | 完成 10 次 72h 冷静期 |
| 七天不败 | 🔥 | 连续 7 天无冲动消费 |
| 月度自律王 | 🏆 | 连续 30 天无冲动消费 |
| 后悔榜常客 | 💸 | 后悔榜上榜 ≥ 3 次（自嘲反向成就） |
| 轻装上阵 | 👜 | 同类替代提醒触发后放弃购买 |
| 临期侦探 | ⏰ | 保质期提醒触发 ≥ 5 次 |
| 一击即中 | 🎯 | 许愿池达成第一个目标 |
| 松鼠收藏家 | 🐿️ | 解锁全部成就 |

### 展示位置
设置页「我的成就」区块，已解锁高亮，未解锁灰色显示进度提示。

### 数据存储
新建 `achievements` 表：
```sql
id, user_id, achievement_key, unlocked_at
```
新建 `user_streak` 表：
```sql
user_id, current_streak, longest_streak, last_check_date
```

## 9. 保质期提醒（Expiry Reminder）

### 概述
食物、化妆品类消费记录保质期，临期前提醒，解决囤货浪费问题。

### 触发条件
记账时 AI 识别到以下分类自动询问保质期：
- 食品饮料
- 美妆护肤
- 保健品

### 录入方式
确认卡片上出现「保质期」可选填项（不强制），用户可填日期或「X 个月」。

### 提醒时机
- 到期前 7 天：主页推送「XX 还有 7 天到期，记得用」
- 到期前 1 天：再次提醒

### 数据存储
`transactions` 表新增字段：
- `expiry_date`: date（保质期）

### 涉及文件
- `src/components/transaction/ConfirmTransactionCard.tsx` — 新增保质期输入
- `src/store/transaction.ts` — 新增到期查询
- 主页推送逻辑新增到期提醒类型
